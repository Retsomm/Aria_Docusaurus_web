開分支之前的**位置**非常重要！

位置：dev

開始做新功能：`git checkout implement/IOTCF-XXXX`

做完要發MR

先到sourceTree

`git commit`

`git push`

到gitLab會看到剛才提交的內容

發MR選擇

目前分支：IOTCF-XXXX

目標分支：dev

成功發MR

等MR合併之後可以在dev看到修改，等到下次要繼續做新功能時，開啟該專案

**記得切換為dev環境**

`git pull `把新增的內容同步到本地端

後續新功能是一樣的流程

如果dev的修改要同步到prod，也是在gitLab上左側欄，點擊MR，會看到新增MR

**目的位置：dev**

**目標位置：prod的分支（可能是master)**

提交MR前要詳述修改內容

合併成功後可以在prod看到修改

剛開始真的很混亂，因為自己在本地端不會有合併MR的情形，因為就是在同一個分支重複地

git add .

git commit -m"XXX"

git push
