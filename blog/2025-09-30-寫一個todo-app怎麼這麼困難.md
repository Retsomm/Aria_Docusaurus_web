我想先說我遇到的困難點是，我認為表單的屬性很困難，有什麼checked、value、type、e.target.value等等。

我還不夠熟悉這些HTML屬性，因為時間都拿來學JS跟React了XD

再來就是JSX的結構，因為我想做的Todo App是可以編輯內容的那種，所以在JSX必須寫判斷。如果是在編輯狀態下，清單就必須在內容區塊呈現input的HTML，給使用者編輯；如果是在非編輯的情況下，清單就只會顯示一般文字的HTML，用於呈現。

**我在手寫程式碼的時候，時常遇到一個問題：什麼時候應該用 {} 跟 () ?**

```
import React from "react";

function App() {
  const names = ["小明", "小華", "小美"];

  return (
    <div>
      <h1>名單</h1>
      <ul>
        {/* 使用 map 直接產生表達式 */}
        {names.map((name, index) => (
          <li key={index}>{name}</li>
        ))}
      </ul>
    </div>
  );
}
export default App;
```

在範例中使用()的地方有五個

1. function(){…}：()在這邊的用意是裝函數的參數。
2. return(…)：()在這邊的用意是裝JSX。當return後的表達式佔多行時，使用()包住可以防止javascript的自動分號插入錯誤。
3. names.map()：()在這邊的用意是裝要傳遞給map的參數，即回呼函數
4. map((name, index)：()在這邊的用意是裝map的參數。
5. .map((name,index)=>(…)：()在這邊的用意是箭頭函數的寫法，如果是單行，可以省略return{}。

在範例中使用{}的地方有兩個

1. function(){…}：{}在這邊的用意是用於定義函數的主體或程式碼區塊。
2. {name.map(…)}：{}在這邊的用意是在JSX中放表達式時要使用{}包住。

真的很複雜對吧！？
